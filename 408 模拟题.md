#### CO
- 指令流水线中 IF、MEM 访存，可能有 TLB/Cache 缺失，而 ID 中的取数、WB 中的写回，都是**针对寄存器堆**的
  只修改寄存器、不访存的指令 MEM 段是空操作
- 一条指令执行过程中，单周期 CPU 的每个控制信号不变，数据通路每个部件只能使用一次；多周期：改变、多次
- ==CPU 执行时间== 只要 CPU 在运行指令，就算入 CPU 执行时间
	- **包括** I/O 调用、进程调度、**内存管理**、文件系统操作
	- **不包括** I/O 等待时间、程序挂起或阻塞等待时间
- 判断 OF 注意次高位的进位是什么，1011+0010 最高位和次高位进位均是 0
- 64M×8位的RAM芯片：每个单元有8位，行数尽量小、行列尽量相同 → **2^13行、列**
  **与地址线是13位进行区分**
- 采用LRU替换的组相联映射可能发生数据刚被替换出又被访问：ABCABC...
- ==(分支)控制冒险== 改变指令执行顺序，转移和返回，中断或异常，有可能产生，不是一定冲突，发生控制冒险的是对应的转移指令等，不是后面被阻塞的！
- 提高 Cache 的“关联度”是指：**增加每个 Cache 组中能容纳的块数量**，从而减少冲突，提高命中率
- 适合采用==异步传输==：速度不同(差异较大)，CPU 和 I/O
- 算两个机器的指令执行速度倍数时，**平均执行时间之比要取倒数**
- 就近舍入：0|1->1，0|0->0
- 寄存器传输指令：寄存器与寄存器之间
- ==运算器== 包含通用寄存器、**状态寄存器PSW**
  ==控制器== 由 PC、IR、CU 组成，取、分析、执行指令
  运算器和控制器集成到芯片上→CPU
- “数据高速缓存”是 Cache
- 一般说寄存器默认是 CPU 中的，Cache 是直接集成在 CPU 中
- ==微指令格式==
	- ==水平型==
		- 直接、字段直接、字段间接编码
		- **一位对应一个控制信号**，格式为**操作控制+顺序控制**（判断测试字段，后续地址字段）
		- 一条水平型微指令定义并执行多个并行操作的微命令
		- **优点**：微程序短、并发能力强、执行速度快；**缺点**：微指令长
		- ==控制存储器容量== 微指令字长24位，减去操作控制字段13位(3+4+4+2四个互斥类)与3位判断测试字段，剩8位下地址字段，共有256条微指令，容量为256×24b
	- ==垂直型==
		- 类似机器指令，设置微操作码，一条垂直型定义并执行一种微命令
		- **优点**：伪指令短、规整、简单、便于编写微程序；
		  **缺点**：微程序长、效率低、速度慢
- 控制存储器中微程序的个数为**机器指令数**+取值间址中断等**公共**微程序数，注意与微指令条数无关（一段微程序若干条微指令）
- 一个时钟周期内可并行传送多次数据
- ==闪存翻译层== 将来自 CPU 的读写请求翻译成对物理设备的读写控制信号，相当于磁盘控制器
  ==闪存芯片== 相当于磁盘中的机械驱动器
  SSD 以**页为单位读/写**，**块为单位擦除**
- ==缓冲存储器== Cache，硬件自动完成，没法地址指定
- CPU 无法直接访问外存，指令只能使用主存地址
- I1 I2 数据冒险默认**不是** WB 前半段写后半段读
- **状态字(状态信息)是数据**，用数据线传，**状态是信号**，用控制线传
- ==硬件多线程== 为每个线程提供**单独**的通用寄存器组和PC等，**线程切换**只需激活选中的寄存器，省去和存储器交换数据
	- ==细粒度多线程== 多个线程间轮流交叉执行指令，能在每个时钟周期切换线程
	- ==粗粒度多线程== 仅在当前进程出现较大开销的阻塞时切换，上述两者**指令级并行，线程级不并行**
	- ==同时多线程 SMT== 同一时钟周期发射不同线程的多条指令，指令级和线程级并行
	  Intel 处理器中的==超线程==是 SMT，在**单核/单处理器**中设置**两套线程状态部件**，**共享高速缓存和功能部件**
- ==多核处理器== 也称片上多处理器，**多个处理单元集成到单个CPU中**，每个核可共享一个 Cache 也可有独立的 Cache。真正意义上并行
- ==共享内存多处理器 SMP== 共享单一物理地址空间的多处理器
	- ==统一存储访问(UMA)多处理器== 访存时间相同，通过集成内存控制器的北桥芯片访存
	- ==非统一存储访问(NUMA)多处理器== 内存控制器集成到 CPU 内部，每个 CPU 独连一部分内存，称为**本地内存**(主存被分配给不同存储器)，CPU 之间通过 **QPI 总线**连接通过 QPI 总线访问其他 CPU 的**远程内存**
- 多核 CPU 的 Cache 一致性还包括各 CPU 的 Cache 之间的一致性，**同一位置的内存数据不同 Cache 的内容应该相同**
- n 个处理机所获得的加速比**小于**一个处理机的n倍
- int转float，可能因为精度而**舍入**
- 地址32位，主存块大小32位，1字32位，全相联，回写，存32K字数据的Cache总容量2048K位，**注意全相联没有行号字段，和直接映射区分**
- 中断字寄存器对程序员不透明，指令缓冲器对程序员透明，移位器、乘法器、先行进位链是ALU的一部分
##### OS
- 第二类虚拟机管理程序运行在用户态
- 引导扇区的内容是安装操作系统时写进去的
- 引起==进程撤销（终止）==
	- 正常结束
	- 异常结束（包含算术运算错、I/O故障）
	- 外界干预：操作员或 OS、父进程请求与终止
- **程序顺序执行**具有可再现性，**多道程序并发**则具有不可再现性
- wati 和 signal 可解决一切**互斥**问题，但不能防止死锁

- DMA 在外设和主存中传数据，同时 CPU 调度执行其他程序，采用 DMA 方式的用户程序被阻塞
- CPU 响应并执行中断时，不一定发生了进程切换，暂停原进程≠切换，“中断处理程序”不是某个进程，它是操作系统内核的一段代码。当前进程无法继续运行时（等IO、缺页等），才阻塞切换进程。中断处理程序结束 _即将返回用户态之前_ 由内核决定进程是否调度和切换
- ==VFS== 
	- 能够把**不同存储介质**（本地磁盘、网络共享、光盘等）和各类文件系统的实现细节**屏蔽**掉
	- 定义了通用文件系统都支持的接口，新文件系统**只要支持这些接口即可安装使用**。抽象了四种对象，每个对象包含数据和函数指针，指向具体文件系统的函数
	- ==超级块对象== 表示一个已安装(挂载)的特定文件系统，对应磁盘上特定扇区的文件系统超级块，存**已安装的文件系统元信息**
		- 操作函数包括分配、销毁、读写 inode
	- ==索引节点对象== 表示一个特定的文件，访问文件时在内存创建索引节点对象
		- 操作函数包括创建新索引节点、创建目录项、创建硬链接
	- ==目录项对象== 是一个路径的组成成分，包含指向关联索引节点的指针、父子目录的指针，其在磁盘上**没有对应的数据结构**，在 VFS 遍历过程中逐个解析为目录项对象
	- ==文件对象== 表示和进程相关已打开的文件(进程视角)，文件对象和物理文件的关系类似进程和程序的关系
		- 包含该文件的目录项文件、文件系统、文件指针，该文件对象上的一系列操作函数
	- 进程调用 VFS 中的一个函数，该函数调用目标文件系统的相应函数，将请求转换为面向物理设备的指令
- ==文件系统挂载== 是指把一个独立的文件系统(设备中)与操作系统的目录结构（即文件树）关联起来，能够通过目录访问该文件系统上的文件，“设备”是逻辑上的设备，一个磁盘不同分区都能视为不同设备
	- 启动时发现所有设备并安装所有文件系统
	- 安装文件系统的目录称为安装点(挂载点)，一个安装点同时只能挂载一个设备，同一个设备可以有多个安装点，安装完成后 OS 通过访问挂载点即可透明地读取、写入该文件系统中的文件，就好像这些文件本来就在本地目录一样-**统一命名空间**
	- 挂载时通过**挂载选项**可以限制对特定文件系统的读写权限
	- **跨文件系统协同**：不同类型的文件系统（如 ext4 与 NTFS）可以并存，用户无需关心底层格式，只需通过统一的路径访问即可
- ==带权周转时间== 作业周转时间/作业实际执行时间
- 死锁预防中破坏请求和保持条件的预先静态分配方法可能会导致饥饿，而仅在运行初期分配并逐步释放的方法则不会
- ==固定分区分配== 内存空间划分为若干**大小固定**的分区，一个分区只装一道作业，分区大小可相等可不等，**无外部碎片**，不能多个进程共享一个主存区
- SPOOLing 技术输入井和输出井在磁盘上
- ==延迟写== 是在内存中设置一个**空闲缓冲区队列**，当其他进程**申请到此缓冲区时**才写回磁盘（是被 “空闲缓冲不足” 触发的）
- 单一连续分配的虚拟地址空间=物理
- 多线程模型是线程-线程，多对一模型在**用户态**管理和切换，一个线程访问内核时阻塞，**整个进程都被阻塞**
- 根据算法访问页框缺页次数要算**初始为空时的缺页**
- ==设备分配与回收== 一个通道连多个控制器，一个控制器连多个设备
	- ==设备控制表 DCT== 每个表项是设备各属性，含设备标识符(物理设备名)，指向控制器表的指针，指向等待该设备的队列指针；释放设备：将 DCT 中的设备状态设置为空闲
	- ==控制器控制表 COCT== 含指向对应通道表的指针
	- ==通道控制表 CHCT== “与通道连接的控制器表首址”表项找到该通道管理的所有控制器信息
	- ==系统设备表 SDT== 整个系统一张，连接到系统的所有物理设备，每个对应一个表目，表目中含有 DCT、驱动程序入口
	- 安全分配：请求IO则进入阻塞态，且不保持资源；不安全分配：请求后继续运行，只有请求的设备被占用时才阻塞，一个进程同时操作多个设备
	- 过程：根据逻辑设备，从SDT中→该类设备DCT，若有空闲→该类设备COCT，若有空闲→该类设备CHCT
	- ==逻辑设备表 LUT== 将逻辑设备名映射为物理设备名，表项为：逻辑设备名、物理设备名、设备驱动程序入口地址，查找LUT找物理设备名和设驱。两种方式：整个系统一张(适用单用户系统)，每个用户一张
- SPOOLing 由输入进程、输出进程和井管理程序管理  
	- 输入/输出进程 负责设备和磁盘井之间的**数据交换**；  
	- 井管理程序 负责用户请求的**管理**、输入井/输出井的**空间分配**和对输入输出进程的**调度**。
- 中断处理程序是IO系统最低的一层
- 一级间接寻址至少给一个进程分配3个页框：指令页+间接地址页+操作数页
- ==分页存储管理== **没有**逻辑上扩充主存容量，解决的是可以不连续存放

- ==虚拟设备== 是将一个物理设备变成多个逻辑设备
