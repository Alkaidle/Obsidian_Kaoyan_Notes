1. “同一台计算机中TCP和UDP分别拥有自己的端口号”：同一台主机上，TCP 和 UDP 各自维护独立的端口号空间；即使端口号数字相同，也不会冲突（五元组识别通信端点）
2. TCP 三次握手时双方告诉对方自己的接口能够适应的 MSS 大小，选较小的（MSS 在三次握手时计算得出）
3. TCP 首部
	- 序号 32 位：$2^{32}$ 个不同的序号
	- 数据偏移：指明的是首部长度，以 4B 为单位，区分 IP 数据报分片中的数据偏移
4. 序号问题
	- **最后一个序号减1**
	- SYN=1、FIN=1 的要消耗一个序号，后面的数据从初始的序号**加1开始**
	- 共发送多少数据，找最后一个数据的字节编号
5. 发送方收到通告的接收窗口，还要减去已发送未确认的数据，得到还能发送的最大字节数
6. TCP 连接状态：看各字段
7. “第 n 个确认段” 默认**连接建立成功后**
8. 拥塞窗口相关问题
	- 慢开始阶段每收到一个确认段，拥塞窗口加1，从一个 RTT  时间来看大小翻倍，实际上仍然是连续增加的
	- “当窗口大小为（上升/达到） n 时发生超时”
		- 当 cwnd = n 时，说明**最多可以有 n 个未被确认的报文段在传输中**（连续发送）
		- 超时是指等待这 n 个中未被确认的某个报文段时超时了

# 6.应用层
1. 域名和 IP 地址不具有一一对应的关系，一个主机可以对应多个域名，一个域名可对应多个主机
2. 提供客户访问的主机不一定有域名，一定有 IP
3. 配置电子邮件用户代理时需要设置服务器的 POP3 地址，以正确接收邮件
4. POP3 协议 ①明文传送不加密，②基于 ASCII 码不能用二进制，③一个账号在服务器上只能有一个邮件接受目录
5. TCP 连接端点由 IP+端口号 确定，故”操作第一步“考虑找服务器的 IP 地址，即域名解析
6. FTP 客户发起对 FTP 服务器连接的第一阶段是建立**控制连接**（控制连接和数据连接都是传输连接，需要进一步区分）
