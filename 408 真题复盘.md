---
- ==线索二叉树==，先序线索树无法通过链域找先序前驱，后序线索树找不到后继
- 主存扩展中，芯片的位是 bit，而地址代表存储单元 1B
- ==RAM 和 ROM==，都是随机访问，很多 ROM 也能像 RAM 一样随机存取（**=随机访问**）
	- DRAM 三种刷新方式，行列复用；SRAM 也是非易失，但不刷新，行列同时送，不复用
	- ==SDRAM== 同步 DRAM，与 CPU 的数据交换同步于时钟信号，**锁存** CPU 发出的地址和控制信号
- 三种方式解决数据冒险，==转发（旁路）技术==，EX 用到前一个 EX 产生的数据，直接从 **EX/MEM** 寄存器中取，中间数据直接转发到 ALU 输入端；==load-use== 不能通过该技术解决，load 在MEM 结束才能得到结果，WB 前半周期写入，use 插入一条 nop，然后在 **MEN/WB** 寄存器中取
- ==控制冒险== 在 MEM 段才能确定 PC 是否改变，插入指令、阻塞/分支预测，延迟损失时间片
- ==中断流程==
	- **中断隐指令**：关中断、保存**断点**、中断服务程序寻址
	- **中断服务程序**：保存**现场**、中断事件处理、恢复现场、开中断、中断返回
- 装填因子=记录数/表长，故要根据记录数算表长；**越大越满**
- 计算线性探测法不成功的 ASL，根据当前的表分别算映射到 0 - k-1 位置上的元素查找失败时的长度，往后找到一个空位置或绕回时为**一次失败查找**，加和/**k**
- C-SCAN 方向一致，默认 LOOK
- 求某个物理地址所在主存块对应的 Cache 号：物理地址写为二进制，找行号对应的位；也可以先找主存块号，再 mod

- 树的题若直接画图不好画，考虑递推，比如平衡二叉树所有非叶结点平衡因子均为1/平衡二叉树比较次数最多/树最高，每一部分都是子问题组合，从最小的问题开始往后递推

- ==循环队列==
	- **rear 指向队尾元素下一个位置**
		- 初始：**front=rear=0**
		- 判空：front=rear
		- 判满：
			- 牺牲一个存储单元：(rear+1)%n=front
			- 增加size：front=rear + size
			- 增加tag，删除 tag=0，插入 tag=1：front=rear + tag
	- **rear 指向队尾元素**
		- 初始：**front=0**，**rear=n-1**
		- 判空：**(rear+1)%n=front**
		- 判满：
			- 牺牲一个存储单元：(rear+**2**)%n=front
			- 增加size：front=rear + size
			- 增加tag，删除 tag=0，插入 tag=1：front=rear + tag
- ==基址寻址== 采用专门寄存器 BR，或通用寄存器--要在指令中增加一段
- ==文件打开== 系统打开表和进程打开表；首次打开从外存中复制表项（文件名-索引节点号）到系统打开表，返回索引号（文件描述符）给用户
- ==索引节点== 文件名和文件描述信息相分离，后者单独形成一个索引节点，文件目录中目录项是**文件名+索引节点号**，FCB 仅是概念上存在
- ==信道利用率== 发送方发送分组到接收到第一个确认分组
	- 连续 ARQ 协议的最大利用率，GBN≥SR，只与发送窗口有关，WT+WR = $2^n$
	- 实际传输速率：利用率×带宽；发送数据量/发送周期
- 找指令某段的值根据该指令的**机器代码**
- **控制相关**是发生转移的**那条指令**
- 算 Cache 命中/缺失率时，算 Cache 块能存项多少数据（例如32B的块能存8条4B的指令）
- 访问 TLB 发生缺页时属于**异常**，故中断后需**再访问一次 TLB**：TLB 访问次数+1/再算一次 TLB 访问时间
  **区分**：Cahche 缺失时在处理过程中 CPU 得到块，不需要再次访问
- **访问磁盘块**
	- 顺序移动考虑向前/向后，每次**修改**访问**2次**，读+写
	- 链式存储找到待修改的结点前：只需读，修改的结点读+写
	- 新写入的结点1次
- I/O 接口中状态寄存器和控制寄存器可共用，传送方向相反且访问时间错开；
  **对状态端口只能读**，**控制端口只能写**
