1. 硬件变换地址机构用于动态重定位，静态重定位用 OS 或装入程序地址变换
2. 页表在进程创建时就**分配在主存**，进程被调度运行时，操作系统把该进程的 **页表基址寄存器（PTBR）** 设置为该进程页表在主存的地址。可以说页表常驻内存能加速虚实地址转换
	- ⚠页表和 PCB 都是常驻内存的
3. 固定分区分配中每个分区大小可以不同但预先固定（确定了不能改变）
4. 地址变换机构（通常由 **基址寄存器** + **界限寄存器** 组成）放在 CPU 的内存管理单元（**MMU**）里
5. 页式系统中*形成有效地址、查页表、拼接物理地址* → 由**硬件**（**MMU**）自动完成
6. ==可重入程序==是通过共享使用同一块存储空间，或通过动态链接将所需的程序段映射到相关进程中，优点是减少了程序段的调入调出，减少了对换数量
7. ⚠区分
	- 对主存的访问：字或字节为单位
	- 对主存的分配：页、段
	- 形成逻辑地址：链接
	- 形成物理地址：装载（静态重定位）、 运行时（动态重定位）
8. 静态重定位是运行前装配程序完成的，要求分配的内存空间必须**连续**
9. 分段中段内位移最大值就是最大段长
10. **二级页表**
	- 页目录只占一页，一个页目录项对应另外另外一页（第二级页表）该页中存页框号
	- 步骤：①先求一页存多少页表项，对应页目录的位数；②页表分多少级：地址除去地位的页内偏移位数/页目录位数，向上取整（每级都跟页目录位数相同）
	- 查页表时先从顶层页表找到对应的二级页表地址（顶层常驻内存），如果二级页表不在内存 → 操作系统分配/调入 → 再完成转换，即**二级页表不一定常驻内存**，节省了内存空间
11. P55 2013年真题
	- 取逻辑地址 LA 高/中/低位的方式："/n"：截取第 n 位之前的高位，"%n"，截取低 n 位
	- 一页 4KB，对于逻辑地址0000 8000H，隐含条件是其是第8个页表项（逻辑地址从0开始）
12. 采用虚拟内存管理方式的系统，malloc() 函数返回的是虚拟地址（在虚拟地址空间的堆区）
13. 虚拟存储器==最大容量==由 **CPU 地址位数决定**，和主存、外存无关，==实际可用容量==受 **主存大小 + 外存（交换空间）大小** 限制，如果程序使用的虚拟内存超出主存+外存之和，就会内存不足，导致进程被终止
14. **驻留集**可定义为
	- “分配给进程的页框集合“（分配视角）
	- “进程当前实际驻留的页面集合”（实际驻留视角）
