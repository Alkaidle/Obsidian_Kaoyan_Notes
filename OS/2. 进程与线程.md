1. 创建进程：用户登录，作业调度（高级调度），系统提供服务，用户程序请求。**设备分配**是在系统中设置数据结构
2. 终止进程时会释放占有的所有资源，包括 PCB，不一定终止子进程
   终止线程时**不立即释放占有的资源**，仍可被其他线程调用，其他线程执行**分离函数**后才释放
3. 单处理器中10个进程，**就绪队列**最多9个，**阻塞队列**最多10个
4. 程序的封闭性指程序执行结果只取决于程序本身，不受外界影响
5. ==CPU 状态==是指 CPU 内部寄存器的信息，也称 CPU 上下文，需要**记录在 PCB 中**
   内核态和用户态是指 ==CPU 运行模式==，用 PSW 记录
6. 父进程和子进程**共享一部分资源**，不共享虚拟地址空间（子程序有自己的）
7. 线程包含 CPU 现场，可以独立执行程序（因为是调度的基本单位）
8. 线程虽然有自己的存储区，但地址空间不是独立的，是属于进程地址空间的
9. 对于键盘这种慢速 I/O 分配一个线程即可
10. 全局变量是相对同一进程而言的，两个进程无法通过全局变量交换数据；可以通过文件系统：管道
11. 用户级 vs 内核级
	1. OS 对于用户级线程调度仍是以进程为基本单位，同一进程内多个线程不能同时调度到多个处理器上运行
	2. 用户级线程由**线程库**在用户空间管理，线程库中线程的切换不会导致进程切换
12. 线程打开的文件属于进程的资源，把文件描述符放进进程的打开文件表中，其它线程可共享，资源不属于线程
13. 多线程模型中的“多对一”是指**一个进程对应一个内核级线程**，访问内核时每次只映射一个线程

### 2.2 CPU 调度
1. 处于（普通）临界区的进程可能因中断或抢占而导致调度；如果临界区请求的是需要等待的资源，如打印机，也需要主动放弃 CPU 产生调度。但是在 OS 临界区中不能被调度。
   调度和临界区是两回事
2. 同一进程中的线程切换时**不需要**更新页基址寄存器和进程打开文件表，因为共享虚拟地址空间和进程打开文件表
3. CPU 繁忙型作业是指该类作业占用很长 CPU 时间而很少请求 I/O，类似长作业
   I/O 繁忙型作业执行时频繁请求 I/O，可能频繁放弃 CPU，占用时间不太长，类似短作业
4. 作业调度
	- 把作业调入内存，就创建进程竞争 CPU，只被调入一次，不涉及“抢占”，”抢占“是抢占 CPU 使用权
	- 内存有空间才执行作业调度算法，一开始按照顺序进入内存，满了则等待执行完
	- 计算作业周转时间用结束时间减去到达时间，”到达“是**到达外存后备队列的时刻**而非被调入内存时
5. 计算周转时间不能算上前面进程的等待时间，只能加它的执行时间、进程切换时间
