2. SQL由硬解析变为软解析的技术称为：绑定变量（参数化SQL）
   思想：将 SQL 语句中的常量替换为变量，使 SQL 结构保持一致，从而复用解析结果和执行计划，减少硬解析次数，提高数据库性能。
```sql
-- 未绑定
SELECT * FROM user WHERE id=1;
SELECT * FROM user WHERE id=2;
-- 绑定
SELECT * FROM user WHERE id=:id;
```
绑定变量可能导致执行计划不适合所有参数：第一个参数全表扫描，导致后面所有的都全表扫描，而有的应该用索引

9. **规范化原则**：通过 1NF、2NF、3NF 减少数据冗余，提高数据一致性。例如将学生信息表和选课表拆分。**适度反规范化原则**：在高并发系统中适当增加冗余字段减少 Join，提高查询性能，例如订单表冗余用户姓名。**索引设计原则**：对高频查询字段建立索引，提高查询效率，但避免过多索引影响写入性能。
   这三者需要在**数据一致性和系统性能之间进行权衡**

15. 给外键建立索引主要是为了 **避免全表扫描、提高 JOIN 性能、减少外键约束检查开销，并降低锁竞争问题**
16. **分区（Partition）** 是指把一张大表按照某个规则拆分成 **多个物理上独立但逻辑上仍属于同一张表的数据块**。分区键类型常见的有 **范围分区、列表分区（按离散取值列表）、哈希分区（对分区键哈希计算）、键值分区（自动使用内部哈希函数）**
17. BCNF：对于关系中的 **每一个函数依赖 X → Y，X 必须是候选键**。                             4NF：在满足 BCNF 的基础上，**消除多值依赖**                                                          5NF：在满足 4NF 的基础上，**消除连接依赖**，保证表能够无损分解且不会产生冗余
18. 单父节点：邻接表模型、物化路径模型、嵌套模型、闭包表模型(单独建一张表记录 **所有祖先—后代关系**) 。多父节点：关系表(parent_id—child_id 多对多)
19. 手动控制并发更新：
	1. **读-修改-写操作**（避免丢失更新）：select ... for update
	2. **跨多记录事务一致性**（如转账）：要么全部成功，要么全部回滚
	3. **乐观锁机制**（版本号字段）
20. 
	1. 实体完整性：每个关系表必须有 **主键**，且主键 **不能为 NULL、不能重复**，用来唯一标识每一条记录。作用：保证每条记录可以被唯一识别，防止数据重复或混乱。
	2. 参照完整性：表中的 **外键** 必须引用另一个表中已经存在的主键值，或者为空。作用：保证不同表之间的数据关系正确，避免出现孤立或不存在的数据引用。
