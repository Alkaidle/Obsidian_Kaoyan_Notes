1. "主存容量 32M×16 位"：MDR、存储字长为 16 位，求主存地址位数要根据总容量以及编制方式
2. CD-ROM 是光盘，串行存取方式（直接存取）
3. 没有说明同时访问 Cache 和 主存，默认 Cache 不命中的时间为两者访问时间之和
4. SRAM、DRAM 引脚问题
	1. 数据引脚数等于存储元位数，如 16×8 位，数据引脚数 8 位
	2. SRAM行列地址同时输入，因此==不存在地址复用==
	3. 由于DRAM采用地址引脚复用，故地址线增加 1 根，容量提高到原来的 4 倍（行列各2倍）
	4. 行、列数是 2 的指数，区分引脚数
5. DRAM 一次刷新需要一次访存，占用一个存储周期
6. 若干 DRAM 芯片构成的存储器刷新时，所有芯片同时刷新，取决于单片的行数，与片数无关
7. "随机访问"与"随机存取"是同样的说法，RAM 和 ROM 都支持
8. 低位交叉编址
	1. 做题时将访存地址（主存地址）→ 模块序号，对地址 mod m（模块数）
	2. 同时启动方式一个存取周期读入所有芯片的同一行，若某变量类型的数据连续存在两行，则需要两次访存（通过地址的低位确定其所在的芯片，再根据其具体类型int、double等以及每片芯片位数确定其所占的芯片数）P28.T33
	3. ==一个模块访问需要一个存储周期，一个存储周期最多读出m个字==
9. MAR 位数决定了==主存地址空间大小==，与主存实际大小无关
10. 磁道（柱面）号、磁头（扇区）号
11. 磁盘存取一个扇区平均时间：平均寻道+平均旋转+扫过一个扇区
12. 磁盘平均旋转延迟时间为 $\frac{1}{2*转速}$
13. 磁盘驱动器就是磁盘（不是 I/O 接口）
14. **Cache**
	1. Cahce 和主存独立编址，Cache 地址空间对 CPU 和程序员完全透明
	2. 以全相联映射为例，"tag+块内地址"的形式是==主存地址==（CPU的视角），由于地址连续，通过 tag 将其分为许多块，对应 Cache 中的块。
	   而 ==Cache 行结构==是"tag+状态位+数据区" ⇒ Cache 行位数（一组对应一行）
	3. 直接映射方式计算标签位数：$\frac{主存容量}{Cache容量}，即2^{m-c}，因为标签位数t=m-c$
	   全相联方式：主存地址位数-块内地址位数
	   组相联方式：主存地址位数-组数位数-块内地址位数
	4. Cache 的存储容量为 $行数×主存块大小$，故主存块大小和 Cache 存储容量无直接联系
	   总容量 = 存储容量 + 标记阵列容量
	5. r 路组相联映射，根据主存地址访问 Cache 时同时进行 r 次标志位比较，每次比较标志位的位数
	6. ⚠ **r 路组数不是 r！**==计算块内地址要根据编址方式==
	7. Cache 地址结构中去除块内地址剩下的表示==主存块号==；若给出十进制主存块号，则可将其表示为二进制，看其地址结构，也可以直接 mod 行数（组数）得出行号（组号）
	8. "先除后余"：访问主存中的 n 号单元时，先根据块大小确定其位于的块数（"/"+向上取整），然后根据 Cache 行数（组数）确定其位于 Cache 的行号（组号）（"%"）。需要注意编号从 0 开始
	   ⚠尤其是==数组单元，根据首地址求出其所在的主存块号(⌊序号/每块容量⌋)==，再映射到 Cache 块号
	   也可以直接根据其地址结构判定
15. 指令和数据都可能涉及时空局部性
16. 虚拟存储器
	1. 缺页时根据页表中（页故障地址）存放的磁盘地址访问磁盘
	2. 分段方式对低级程序员（用段号编程）和编译器（用段号链接）都是==不透明==！
	3. 相联存储器：按内容（关键字）查找，硬件级并行，Cache tag 和 TLB
	4. 缺页处理（主存-外存替换）由软件完成，缺页异常处理程序
