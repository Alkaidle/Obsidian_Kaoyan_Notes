1. PC
	1. PC 的值在 CPU 执行指令过程中（取指令）修改
	2. 主存按字节编址，PC 使用字节编址则位数等于存储器位数；若按字边界对齐，指令起始地址必须是字长的整数倍
2. 取指操作是控制器自动进行，不需相应的指令
3. 不同长度的指令（单字指令、双字指令等）取值操作不同
4. 空操作指令的指令周期中任何寄存器内容都不改变×（PC 自动加 1）
5. CPU 数据通路基本结构分为：总线结构（单、双、多）和专用数据通路结构（不采用内部总线结构）
6. 数据通路是在功能部件间传送的路径，由控制部件控制，不包含控制部件
7. 单周期 CPU 一个时钟周期内数据通路上每个功能部件只使用一次，控制信号取值一直不变
8. CPU 读写控制信号不仅能控制内部数据流方向，还能控制外部（→数据总线→存储器）的数据流
9. **断点**
	- 故障若返回，则返回当前发生故障指令；==也有可能不返回原断点==（非法操作码、除数为0）
	- 自陷返回到下一条指令
	- 终止无法确定有效断点
	- 中断返回下一条指令（因为是当前指令执行完再检查的）
10. 异常/终端**响应**由硬件实现，**响应后**执行的是 OS 内核程序
11. CPU 控制流：CPU 所执行指令的地址序列，正常控制流是指：顺序执行指令或通过转移指令得到的控制流
12. 存储保护错：访问的地址超出了当前进程被分配的合法地址范围。如地址越界、权限错误、用户态进程访问内核态专用区域
13. 微处理器与采用微程序控制器无直接关系
14. **CPU 周期**：指完成一次最基本的 CPU 操作所需的时间，也称机器周期、工作周期
    **指令周期**：取出并执行一条指令的时间，可分为取指、间址、执行、中断周期
15. 断定方式：根据后继地址字段直接指出或转移控制字段控制产生
16. μPC 位数表示控制存储器能存多少条微指令，μIR 位数表示一条微指令长度
    **控制存储器容量**计算类比主存
17. CPU 能理解并直接执行**微指令**和机器指令
18. 微指令设计中“可判定的外部条件”用于条件测试字段，同样需加一个不转移的情况
19. 写微操作命令以及节拍安排，节拍用 T + 下标标明
20. **指令流水线**
	1. 上一条指令在 ID 段阻塞，则下一条指令应在 IF 段阻塞，否则会取值覆盖 PC 内容导致上一条指令译码结果出错
	2. 采用转发技术解决数据相关不用任何推迟
	3. 采用==延迟加转发==处理 load-use，只用阻塞一个时钟周期

21. **多处理器**
	1. 多处理器/硬件多线程/多核处理器/共享内存多处理器
	2. 阵列处理机属于 SIMD：一个计算阵列，包含多个处理单元
	3. 标量流水线处理属于 SISD：一段时间执行一条指令
	4. 超线程技术是单核/单处理器，复制必要的线程资源让两个线程同时进行来模拟多核
	5. 多任务系统/多道程序系统：宏观并行，微观串行，可运行在单核 CPU
	6. 多核处理器集成的是执行核心
	7. 硬件多线程技术是使一个核处理多个线程
