1. **I/O 接口**
	1. 独立编址 存储单元和 I/O 设备仍使用相同地址线，靠不同编址策略和控制信号区分，该方式下 CPU 需要设置专门的输入/输出指令访问端口
	2. I/O 指令是指令系统一部分，是机器指令的一类，格式和通用指令不同
	3. CPU 和主存 -- I/O 总线 -- I/O 接口 -- 通信总线 -- 外设
	4. I/O 接口中数据缓冲寄存器、命令/状态寄存器的内容都是通过数据线传送（三者是 I/O 接口中主要的寄存器）
	5. I/O 指令实现的数据传送通常发生在 CPU 寄存器和 I/O 端口之间
2. 注意==中断向量==是中断服务程序的入口地址
3. 异常和中断通常视为中断，除非专门区分两者的题目
4. Cache 缺失由硬件自动处理，不涉及中断
   虚拟存储器失效产生缺页异常，提出缺页中断
   浮点数运算上溢属于异常，也可提出中断
5. CPU 无法对发生的中断请求立即处理（需要在指令执行结束时响应），进而在 I/O 接口中设置INTR 保存中断请求
6. 对于多重中断，屏蔽字可改变**中断服务程序执行完的次序**，"**开始执行的顺序**"是由响应优先级决定的。中断处理优先级决定了本中断**能否打断正在执行的中断服务程序**
   开中断时 CPU 检测到中断请求则立即**响应**
7. 磁盘寻道和旋转结束后都会以**中断**方式通知
8. 注意区分==保存断点==（中断隐指令，硬件自动）和 ==保存现场、通用寄存器==（中断服务程序→软件）
9. 外部设备中断请求通常是为了 I/O，通常不能发出不可屏蔽中断（影响安全和稳定）
10. DMA 传送前由**设备驱动程序**设置传送参数
11. 若外设准备数据的时间小于中断处理的时间，则写入缓冲区的速度快于读缓冲区的速度，可能造成数据覆盖
12. **计算题**
	1. 计算设备 I/O 占 CPU 总时间百分比：算每秒传输的次数（每秒传输数据/接口一次传的数据），次数×每次的时钟周期数/ CPU 主频，注意 ==DMA 方式==只有**预处理和后处理**需要 CPU 参与，中间的传输时间不计
	2. 数据传输时间不能小于中断处理时间：否则数据被刷新造成数据丢失
