#### 1-HCI Introduction
1. 人机交互 (HCI) 的定义及人-机-交互概念解释
• **定义**：研究人类如何与计算机系统进行交互的学科，它涵盖了人们与计算系统交互的所有方面。其核心目标是设计出能让用户以**最小的努力和挫折感**进行操作的界面。

• **概念拆解**：
    ◦ **人 (Human)**：指系统的用户。可以是单个用户，也可以是共同工作的一组用户。
    ◦ **机 (Machine)**：指用户使用的任何计算技术或设备，如移动电话、电脑、嵌入式系统等。
    ◦ **交互 (Interaction)**：指人与机器之间为了完成某项任务而进行的任何信息交换或沟通。

2. 人机交互的交叉学科特性

• 人机交互是一个高度跨学科的领域，它融合了多种学科的知识，包括但不限于：**计算机科学**（技术实现）、**心理学**（人类感知与认知）、**社会学**、**设计学**（视觉与交互流程）、**人体工程学**（物理适配）以及**人类学**等1。

3. 以用户为中心的设计 (User-Centered Design, UCD)

• **什么是 UCD**：UCD 是一种设计流程和哲学，它要求在设计的每一个阶段都以**用户**及其需求、目标、任务和环境为核心1。

• **UCD 的六大准则 (Six "Mantras")**
**了解用户**（Know the User）
**基于用户数据进行设计**，而非凭空猜测。
**设计师不代表用户**（You are not the user）。
**制作原型**（Prototype）。
**评估**（Evaluate）。
**不断迭代**（Iterate）。

• **为什么 UCD 很重要？**：因为系统是为人设计的。如果不遵循 UCD，可能会导致设计的系统无法满足真实需求、用户使用困难，甚至导致产品失败。UCD 确保了产品不仅功能正确，而且**可用性**强1。

4. 用户体验 (UX) 设计及 UI 与 UX 的区别

• **用户体验 (UX) 设计**：是一个创造产品以提供有意义且相关的体验的过程。它不仅关注界面的美观，还关注获取、整合产品过程中的功能、易用性和效率1。

• **UI (用户界面) 与 UX (用户体验) 的区别**：

    ◦ **UI (User Interface)**：是人机交互发生的物理或数字化**空间/表面**（如按钮、颜色、布局）1。

    ◦ **UX (User Experience)**：是用户在使用产品过程中的**整体感受和过程**1。

    ◦ **关系**：好的 HCI 通常由“好的 UI + 好的 UX”共同构成1。

5. 用户参与闭环 (User in the loop)

• 这一概念强调在整个设计和开发周期中，必须有**真实用户参与**。通过不断的**评估（Evaluation）**获取用户反馈，并将其作为改进设计的依据，形成一个“设计-评估-修改”的闭环，确保最终产出符合用户预期1。
